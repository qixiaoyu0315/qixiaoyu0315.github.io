Ⅰ、五大数据类型

一、 String

1.1 底层实现

  • Redis字符串 为 简单动态字 符串（simple dynamic string，SDS）的抽象类型

    Redis字符串以键值对存下

    下列命令实际创建了 msg 的SDS 和 “hello world”的SDS

    SDS用途：

    • 来保存数据库中的字符串值
    • 用作缓冲区 （buffer）：AOF缓冲区，以及客户端状态中的输入缓冲区

  • 数据结构

    图中 除存Redis之外还有 ‘\0’ 此为SDS函数自动完成 ，遵循空字符结尾这一 惯例的好处是，SDS可以直接重用一部分C字符串函数库里面的函数。

    • len 表示以用长度 不包括空字符 可以不遍历字符串就获得其长度信息
    • free 表示未使用的空间 可以修改 拼接等时使用 提高效率
      • 在strcat拼接字符串时 先判断free是否够用 不够则先扩容，防止字符串溢出

  • 扩容机制

    • len小于1M时，len与free保持一致 总长度 len+free+1
    • len大于1M时，每次扩大1M 总长度len+1M+1
    • 字符串最大容量512M

1.2 命令

  1. dump key 序列化key

  2. exists key 判断keyy是否存在

  3. expire key seconds 设置过期时间

    expireAT 设置 参数是 UNIX 时间戳(unix timestamp)

    pexpire 毫秒设置

    pexpireAT 毫秒 时间戳

  4. persist key 移除过期时间

  5. ttl key | Pttl key 剩余过期时间

  6. move key db 将当前数据库的 key 移动到给定的数据库 db 当中

  7. randomKey 当然数据库随机返回一个key

  8. Rename key newkey 重命名

  9. RenameNX key newkey 仅当 newkey 不存在时，将 key 改名为 newkey

  10. scan cursor [MATCH pattern] [COUNT count] 迭代数据库中的数据库键。

  • cursor - 游标。
  • pattern - 匹配的模式。
  • count - 指定从数据集里返回多少元素，默认值为 10 。

SCAN 返回一个包含两个元素的数组， 第一个元素是用于进行下一次迭代的新游标， 而第二个元素则是一个数组， 这个数组中包含了所有被迭代的元素。如果新游标返回 0 表示迭代已结束。

  1. type key 存储类型
  2. getrange 指定获取 setrange 指定位置追加 append key value 末尾追加

  1. setnx msetnx

  2. setex Psetex

  3. incr incrBy key increment（增量） incrByFloat key increment（浮点增量）

  4. decr decrBy key decrement
